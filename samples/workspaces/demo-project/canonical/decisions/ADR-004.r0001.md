## Context
Milestones 1 through 8 established Memora's local-first core, but the next
phase needed a stable execution order that reflects both current repo reality
and the real MVP bar for external use.

## Decision
Phase 2 proceeds in this order:

1. M9 - Deterministic Project State Model
2. M10 - External Workflow, Codex First
3. M11 - Human Review UI
4. M12 - Existing Repo Understanding MVP
5. M13 - Mobile Contribution MVP

This roadmap lives first in private Memora-style planning artifacts. Public
repo docs remain focused on implemented behavior and current-state references.

## Alternatives Considered
- keep the earlier milestone names and broader scopes unchanged
- move mobile earlier even though import and review flows are still unsettled
- use public docs as the primary roadmap store

## Consequences
The roadmap now has an explicit dependency chain. M9 becomes the immediate next
execution target, M10 becomes the next milestone to prepare at implementation
grade, and M11 through M13 remain intentionally revisitable after earlier
milestones land.
