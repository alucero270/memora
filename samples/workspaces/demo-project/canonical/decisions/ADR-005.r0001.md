## Context
The repo already contains deterministic context assembly, inclusion reasoning,
ranking, caching, and runtime-facing contract shapes. The next milestone should
stabilize that existing projection instead of inventing a new engine.

## Decision
Milestone 9 is the immediate next execution milestone. It formalizes and
stabilizes the deterministic project-state projection without introducing a
parallel project-state model.

Milestone 10 is the next milestone to prepare at execution-ready depth. It
should be detailed enough for issue creation now, but it still waits for M9
completion before implementation.

## Alternatives Considered
- start with external workflow and adjust the read model later
- fold M9 and M10 together into a single wider milestone
- delay M10 packet writing until after M9 implementation

## Consequences
The planning baseline now distinguishes between immediate execution, next-ready
execution, and later draft packets. This reduces churn in M9 while still
keeping momentum for M10.
