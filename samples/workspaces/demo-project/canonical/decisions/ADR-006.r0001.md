## Context
Memora needs to support Codex, ChatGPT, and Claude-oriented workflows without
splitting the core contract by vendor or burning rate limits through repeated
manual context loading.

## Decision
Memora continues with an MCP-first shared interface and companion OpenAPI
surface. Codex is the first operational external client for M10. ChatGPT is
treated as a read and validation client, while Claude remains part of the
shared design target even if it is not the first fully exercised client.

## Alternatives Considered
- build separate vendor-specific paths from the start
- make ChatGPT or Claude the first operational client instead of Codex
- defer shared contract alignment until after mobile planning

## Consequences
M10 must stay contract-first. Later milestone packets must avoid vendor-specific
branching unless real implementation constraints prove it necessary.
