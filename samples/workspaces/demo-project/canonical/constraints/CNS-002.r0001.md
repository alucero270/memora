## Constraint
Milestone 9 may formalize and serialize the existing deterministic context
projection, but it must not invent a parallel domain model or a second source
of project-state truth.

## Why It Exists
The repo already has context assembly and runtime contract shapes. A duplicate
state model would increase churn, invite drift, and undermine deterministic
behavior.

## Implications
M9 packets should talk about projection schema, serialized output,
deterministic tests, and existing contract alignment. They should not describe
a new engine or standalone truth model.
