## Constraint
The first mobile slice must remain local-first and file or template driven. It
may support local storage, copy/export flows, and a Nextcloud-style shared
folder bridge, but it must not require MCP, hosted services, live sync, or a
mobile approval workflow.

## Why It Exists
Mobile contribution is valuable, but sync complexity can easily overwhelm the
core roadmap if it enters too early.

## Implications
M13 packets should bias toward structured capture and desktop import. Same
network sync remains a later question, not an immediate implementation
requirement.
