## Question
Which retrieval aids could eventually support Memora without changing canonical
truth or deterministic final context assembly?

## Context
The project wants to preserve useful future-track thinking, but it does not
want speculative ideas to blur current MVP boundaries or appear as approved
core behavior.

## Possible Directions
- external semantic candidate discovery beside Memora rather than inside core
- a small internal model for query rewriting, reranking, or summarizing
  retrieved candidates
- a later backfill/import workflow for large existing repositories

## Resolution
Deferred. These ideas belong to a future track only. They may be explored later
if they remain advisory, preserve filesystem truth, and keep approval-governed
Memora artifacts as the canonical source of project memory.
