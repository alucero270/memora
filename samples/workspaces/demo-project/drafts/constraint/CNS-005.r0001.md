## Constraint
Milestone 10 must continue using the existing shared Memora-facing contract and
must not add provider-specific business logic to core services or introduce a
second runtime-facing state model.

## Why It Exists
M10 is meant to prove one real external workflow on top of the current Memora
boundary. If the milestone expands into provider-specific core logic or
duplicate read-model behavior, it will weaken the architecture that M9 just
clarified.

## Implications
Codex and ChatGPT paths should adapt to the shared contract. Differences in
client capability should be handled in adapters, workflow docs, or validation
notes rather than in provider-specific core behavior.
