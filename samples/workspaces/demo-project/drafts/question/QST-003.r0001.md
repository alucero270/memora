## Question
Should the first formalized project-state projection include complete artifact
bodies by default, or should it prefer summaries and only expand bodies when
needed?

## Context
The current repo shape can already carry full artifact content, but larger
repositories may make that expensive if the default projection becomes too
wide.

## Possible Directions
- keep full bodies first and optimize only when size becomes a real problem
- use summaries by default and add explicit expansion rules
- mix the two based on artifact type or focus context

## Resolution
Open. The current baseline leans toward full bodies until a proven size problem
appears.
