## Question
How should Memora-assisted agent workflows proactively surface when durable
decisions, constraints, outcomes, or questions should be captured and when the
user should revisit them for approval?

## Context
The current workflow can complete useful work, but it still relies on the user
to remind the agent to record important decisions or to validate newly added
artifacts. That weakens usability and makes durable-memory capture feel
optional instead of integrated.

## Possible Directions
- add a repeatable workflow checkpoint that asks whether the turn produced
  durable project memory worth recording
- surface a lightweight review prompt when draft artifacts accumulate or when a
  decision changes milestone execution scope
- add automated artifact-parse validation whenever new reviewable artifacts are
  created
- distinguish clearly between transient execution notes and candidate durable
  artifacts before the turn closes

## Resolution
Open. The usability gap is now tracked explicitly, but the preferred workflow
solution still needs design and validation.
