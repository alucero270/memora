## Question
Should the mobile contribution packet be Markdown, JSON, or a dual-format
structure that supports both human copy/paste and later desktop import?

## Context
The mobile slice should stay local-first and low-friction, but it also needs a
shape that desktop Memora can ingest without ambiguous meaning.

## Possible Directions
- Markdown only for maximum human readability
- JSON only for cleaner import behavior
- Markdown plus an optional JSON payload for future machine validation

## Resolution
Open. The current baseline leans toward a dual-format packet.
