## What Happened
During current workflow use, the user needed to remind the agent to capture
durable decisions and to validate or parse newly added artifacts instead of the
workflow surfacing those steps consistently on its own.

## Why
The current workflow still depends too much on the agent recognizing when a
conversation has crossed the line from transient execution chatter into durable
project memory that should be recorded or reviewed.

## Impact
This makes the workflow less trustworthy and more cognitively expensive for the
operator. Important decisions can remain implicit unless the user explicitly
asks for Memora-style artifact capture.

## Follow-up
Track the usability problem explicitly and design a clearer workflow for:

- surfacing when a decision, constraint, outcome, or question should be recorded
- prompting for review or approval timing
- validating that newly added artifacts actually parse and fit the governed
  model
- making artifact review and approval available from the coding environment so
  Memora does not require a separate approval workflow that operators will avoid
