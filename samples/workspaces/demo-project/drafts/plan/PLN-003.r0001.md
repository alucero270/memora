## Goal
Make Memora artifact review usable from the IDE so an operator can inspect,
approve, or reject reviewable artifacts without leaving the coding workflow.

## Scope
- start with VS Code and Cursor
- treat Cursor as VS Code-compatible unless implementation evidence says
  otherwise
- keep the IDE layer as a review surface, not a second governance system
- preserve filesystem-first truth, lifecycle enforcement, and approval rules

## Acceptance Criteria
- M11 includes explicit VS Code and Cursor review issues
- the IDE review boundary preserves Memora lifecycle and approval rules
- the first IDE surface can list and preview reviewable artifacts
- IDE approve and reject actions route through governed Memora workflow

## Issue Sequence
1. M11-05 - Define VS Code and Cursor review boundary (#195)
2. M11-06 - Add VS Code and Cursor artifact review inbox (#196)
3. M11-07 - Bridge IDE approve and reject actions to governed workflow (#197)

## M11-05 Boundary
Purpose: decide the smallest IDE integration boundary before implementation.

Acceptance focus:
- define what the IDE may display and invoke
- identify the shared Memora path the IDE should use
- document that the IDE must not directly write canonical artifacts

## M11-06 Review Inbox
Purpose: let the operator see reviewable artifacts from VS Code or Cursor.

Acceptance focus:
- list reviewable non-canonical artifacts
- show artifact identity, metadata, validation state, and preview content
- open the backing artifact file when useful
- reuse existing Memora state and artifact behavior

## M11-07 Approve And Reject Bridge
Purpose: let the operator act on artifacts from the IDE without bypassing
governance.

Acceptance focus:
- approve through Memora lifecycle rules
- reject or leave artifacts non-canonical through the governed workflow
- show validation and lifecycle failures clearly
- preserve canonical writes as Memora-owned persistence behavior

## Notes
The operator UI remains valuable, but the IDE path is not optional polish. It
addresses the core usability problem that Memora review must fit inside the
place where project work is already happening.
