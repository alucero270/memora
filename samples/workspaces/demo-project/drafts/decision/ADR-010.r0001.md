## Context
The operator found that reviewing and approving Memora artifacts through a
separate workflow would make the tool too expensive to use during real
development. VS Code and Cursor are already where coding review and agent work
happen, so M11 review needs to meet the operator there.

## Decision
Treat VS Code and Cursor as first-class M11 review surfaces for viewing
reviewable artifacts and approving or rejecting them through Memora's governed
workflow.

The IDE surface must reuse Memora's existing lifecycle and persistence rules.
It may make review easier, but it must not become a second source of truth or a
direct canonical write path.

## Alternatives Considered
- require all review through a separate operator UI
- defer IDE review until after M11
- allow IDE tooling to edit canonical artifacts directly

## Consequences
M11 should include an IDE review path alongside the operator UI work. The first
implementation should stay narrow: artifact inbox, artifact preview, and
approve/reject actions routed through the same governed workflow as the rest of
Memora.
