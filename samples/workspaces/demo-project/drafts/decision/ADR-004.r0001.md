## Context
Milestone 9 clarified the deterministic state-view shape, the contract fit, and
the boundary between canonical and non-canonical state. The next milestone
needs a thin executable slice rather than another umbrella planning pass.

## Decision
Treat the current Milestone 10 issue set as the executable M10 slice:

- M10-01 / #144
- M10-02 / #148
- M10-03 / #149
- M10-04 / #145
- M10-05 / #153
- M10-06 / #154
- M10-07 / #152

## Alternatives Considered
- use umbrella issue #124 as the main implementation issue
- add new M10 issues before execution starts
- defer milestone execution until the issue packet is rewritten again

## Consequences
M10 remains narrowly executable after M9. Review and implementation work can
track the existing issue stack instead of splitting effort across duplicate or
overlapping planning surfaces.
