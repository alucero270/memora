# Change Order — Memora System Evolution

## Change ID
CO-001

## Date

## Summary

Memora is evolving from a structured artifact storage and validation system into a **System of Understanding for AI-assisted development**.

## Context

AI-assisted development is producing systems where:
- code is generated rapidly
- comprehension is not guaranteed
- traceability is lost
- ownership becomes unclear

This results in “dark code” — systems that function but are not understood.

Memora addresses this by ensuring:
- understanding exists before code
- understanding persists after code
- understanding is structured, traceable, and queryable

## Decision

Memora will expand responsibilities to include:

### 1. Source-of-truth artifacts
- plans (specs)
- requirements (FR/NFR)
- decisions (ADRs)
- components

### 2. Structured understanding
- typed relationships between artifacts
- dependency and impact tracking
- traceability across system layers

### 3. Context assembly
- deterministic context bundles
- task-oriented understanding sets
- explanation of inclusion (why something is relevant)

### 4. Understanding outputs
- human-readable system views
- traceability views
- component understanding
- requirement impact views

## Explicit Non-Goals

Memora will NOT:
- perform raw document retrieval (Strata responsibility)
- execute workflows or orchestration
- generate or modify code automatically
- act as a general-purpose knowledge base
- replace graph engines or vector stores

## Impact

### Unchanged
- artifact schema and validation (Milestone 1)
- lifecycle enforcement
- filesystem source of truth

### Expanded
- relationship modeling and persistence
- traceability support
- context assembly logic
- output generation

### Affected Milestones
- Milestone 2 — expand into relationship + traceability layer
- Milestone 3 — redefine as context assembly core
- Milestone 4+ — shift toward outputs and integration surfaces

## Risks Introduced
- system complexity increase
- risk of becoming monolithic
- unclear boundaries with Strata or external graph systems

## Mitigation
- strict module boundaries
- incremental implementation
- deterministic behavior rules
- minimal relationship types initially
