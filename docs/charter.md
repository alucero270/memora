# Charter

## Name

Memora

## Problem Statement

AI-assisted software development breaks down across sessions because important
project state is trapped in chat, prompts, or human memory. Plans, decisions,
constraints, repository understanding, and outcomes drift or are lost, causing
repeated re-explanation, inconsistent execution, and hallucinated continuity.

## Users

- solo developers using AI heavily
- AI agents acting as development collaborators
- future collaborators and teams later

## Why Now

AI tools are improving execution quality, but they still do not provide
durable, structured, approval-governed project memory. As projects become more
agentic, the need for explicit memory, lifecycle control, and grounded context
becomes more important, not less.

## Definition Of Success

- a project can be reopened after time away with immediate state clarity
- planning can be promoted into structured memory
- agents act against approved project context rather than prompt-only memory
- important decisions and outcomes remain traceable over time
- canonical memory stays human-inspectable and approval-governed

## Product Identity

Memora is a local-first structured memory and governance system for
AI-assisted software development.

Memora is not:

- a chat history store
- a generic knowledge base
- a vector database
- an execution runtime
- a replacement for Strata

## Core Promise

Memora preserves structured, approved, durable project cognition across
planning and execution workflows.
