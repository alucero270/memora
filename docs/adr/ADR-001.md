# ADR-001: Memora as System of Understanding

## Status
Accepted

## Context

AI-assisted development has introduced a class of systems where:
- code is generated faster than it is understood
- authorship is distributed across humans and agents
- comprehension is no longer guaranteed

Traditional responses (observability, pipelines, testing) do not solve the core issue:
→ lack of durable system understanding

Memora was originally designed as an artifact storage and validation system.

## Decision

Memora will evolve into a **System of Understanding**, responsible for:

- defining and storing structured intent (specs, requirements, ADRs)
- modeling system structure (components, relationships)
- maintaining traceability across artifacts
- assembling deterministic context for agents
- producing human-readable understanding outputs

## Consequences

### Positive
- Enables comprehension-first development
- Prevents “dark code” accumulation
- Provides long-term memory for agents
- Creates explainable system state

### Negative
- Increased scope and complexity
- Requires careful boundary management
- Risk of overlapping with graph or retrieval systems

## Alternatives Considered

### Keep Memora as artifact store
Rejected — does not address understanding gap

### Use external graph system (Graphiti) as core
Deferred — may integrate later, but Memora must own meaning and validation

### Combine with Strata
Rejected — violates separation of concerns (retrieval vs understanding)
